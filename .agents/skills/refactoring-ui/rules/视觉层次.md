# 视觉层次优化规则

视觉层次是让界面「看起来被设计过」的最有效工具。核心思路：刻意弱化次要信息，突出最重要元素。

---

## 规则 1：主次分明用字重，不用字号

**问题**：过度依赖字号控制层次 → 主要内容过大，次要内容过小。

**做法**：
- 主要元素用 `font-weight: 600` 或 `700`，保持合理字号
- 次要文字用更柔和的颜色，不用极小字号
- 三级内容用最浅色，不用超细字重（`< 400`）

**Tailwind 示例**：
```html
<!-- ❌ 错误：超大字号区分主次 -->
<h1 class="text-4xl">标题</h1>
<p class="text-xs text-gray-500">辅助文字太小</p>

<!-- ✅ 正确：字重 + 颜色区分 -->
<h1 class="text-xl font-semibold">标题</h1>
<p class="text-sm text-gray-500">辅助文字用柔和颜色</p>
```

**核心原则**：两种字重就够 UI 用——常规（400/500）和加粗（600/700）。远离 400 以下的字重。

---

## 规则 2：坚持两到三种文字颜色

- **深色**（主要内容，如文章标题）：`text-gray-900` 或接近纯黑
- **灰色**（次要内容，如发布日期）：`text-gray-500`
- **更浅灰**（三级内容，如版权声明）：`text-gray-400`

不要引入第 4 种文字颜色——保持层次清晰。

---

## 规则 3：有色背景上不用灰色文字

**问题**：在白色背景上，`text-gray-400` 效果良好（降低对比度）。但在有色背景上（如蓝色导航栏），灰色文字会显得褪色、发暗，像被禁用了一样。

**错误做法**：
```css
/* ❌ 有色背景上的灰色文字看起来褪色 */
background: #3b82f6;
color: #9ca3af; /* 灰色 */
```

**正确做法**：手动选择基于背景色的颜色。保持相同色相，调整饱和度和明度直到合适：
```css
/* ✅ 基于背景色相挑选文字颜色 */
background: hsl(217, 91%, 60%); /* 蓝色背景 */
color: hsl(217, 90%, 85%);      /* 浅蓝文字——色相相同 */
```

**白色文字降低不透明度也会褪色**：
```css
/* ❌ 半透明白色在有色背景上显得灰暗 */
color: rgba(255, 255, 255, 0.7);

/* ✅ 手动挑选基于背景的颜色 */
color: hsl(217, 70%, 80%);
```

---

## 规则 4：通过弱化来强调（反衬法）

**问题**：主要元素不够突出，但你没有什么可以添加来增强它。

**做法**：不要进一步强调目标元素——而是弱化与它竞争的元素。

**场景 1：激活导航项不够突出**
```html
<!-- ❌ 试图让激活项更突出但效果不佳 -->
<a class="bg-blue-500 text-white">激活项</a>
<a class="text-gray-700">非激活项</a>

<!-- ✅ 弱化非激活项 -->
<a class="text-gray-900 font-medium">激活项</a>
<a class="text-gray-400">非激活项</a>
```

**场景 2：侧边栏与主内容竞争**
```html
<!-- ❌ 侧边栏有背景色，与主内容竞争注意力 -->
<aside class="bg-gray-100 p-4">...</aside>
<main class="p-4">...</main>

<!-- ✅ 移除侧边栏背景色，让内容直接放在页面背景上 -->
<aside class="p-4">...</aside>
<main class="p-4">...</main>
```

---

## 规则 5：不要执着于「语义化」颜色

**问题**：为了用绿色表示「积极」，给整个元素/卡片绿色背景。绿色可以辅助传达信息，但不要让颜色主导设计。

```html
<!-- ❌ 绿色背景过于抢眼 -->
<div class="bg-green-100 border border-green-500 p-4 rounded">
  <p class="text-green-800">+15% 本月增长</p>
</div>

<!-- ✅ 颜色辅助，不主导 -->
<div class="p-4">
  <p>本月增长 <span class="text-green-600 font-semibold">+15%</span></p>
</div>
```

---

## 检查清单

执行视觉层次优化时，逐项检查：

- [ ] 主要文字是否用字重（600/700）而非超大字号突出？
- [ ] 是否只有 2-3 种文字颜色（深色/灰色/浅灰）？
- [ ] 有色背景上的文字是否基于背景色相手动挑选？
- [ ] 是否通过弱化竞争元素来反衬主要元素？
- [ ] 语义化颜色是否只做辅助，不主导设计？
- [ ] 是否有不必要的背景色在争夺注意力？
