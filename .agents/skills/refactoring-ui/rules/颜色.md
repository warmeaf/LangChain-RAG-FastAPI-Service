# 颜色优化规则

颜色是最直观的视觉效果，但也是最容易被滥用的。好的颜色系统让界面和谐、可访问、易于维护。

---

## 规则 1：用 HSL 而非 Hex

**Hex 的问题**：视觉上相似的颜色在代码中看起来完全不同。`#3b82f6` 和 `#60a5fa` 都是蓝色，但 hex 值看不出来。

**HSL**（`hsl(色相, 饱和度%, 明度%)`）用人类直觉表达颜色：
- **色相 (Hue)** — 0-360°，颜色在色轮上的位置（0=红，120=绿，240=蓝）
- **饱和度 (Saturation)** — 0-100%，0%=灰色，100%=鲜艳
- **明度 (Lightness)** — 0-100%，0%=黑，50%=纯色，100%=白

```css
/* ❌ Hex 值看不出关系 */
color-1: #3b82f6;
color-2: #60a5fa;

/* ✅ HSL 直观表达颜色关系 */
color-1: hsl(217, 91%, 60%);
color-2: hsl(217, 91%, 68%);  /* 同一色相，明度不同 */
```

**调整颜色的操作变得直观**：
- 更暗？降低 L
- 更亮？提高 L
- 更鲜艳？提高 S
- 偏暖/偏冷？旋转 H

浏览器完全支持 `hsl()` 和 `hsl()` 带 alpha 的 `hsla()`。

---

## 规则 2：你需要的颜色比你想象的多

五种 hex 代码建不了任何东西。一个完整的调色板需要：

**灰色系**（8-10 个色阶）
- 文字、背景、面板、表单控件——界面中几乎所有东西都是灰色的
- 从非常深的灰（非纯黑 `#000`）逐步过渡到白

**主色**（5-10 个色阶）
- 按钮、激活导航、链接等主要操作的颜色
- 需要深浅变体：浅色做背景，深色做文字

**强调色**（每种 5-10 个色阶）
- 红色：破坏性操作（删除按钮、错误消息）
- 黄色/橙色：警告消息
- 绿色：积极趋势、成功消息
- 可能还需要：区分不同类别元素的颜色（图表线条、标签颜色）

**总计**：复杂 UI 可能需要多达 10 种颜色，每种 5-10 个色阶。不要怕多——实际使用中你会发现 3-4 个灰色根本不够。

**不要用纯黑 `#000`**——用非常深的灰色，如 `hsl(0, 0%, 13%)` 或更微妙。

---

## 规则 3：提前定义色阶，不要动态生成

**为什么不用 Sass 的 `lighten()`/`darken()`**：你会得到 35 个看起来一模一样的略有不同的蓝色。

**手工构建色阶的方法**：

1. **选基础色**（500）— 作为按钮背景色会很合适的颜色
2. **先选两端**：
   - 最深色（900）— 通常用于文字
   - 最浅色（100）— 用于有色背景
3. **填补中间色阶**（900 → 700 → 500 → 300 → 100）：
   - 选 700 = 500 和 900 的中间色
   - 选 300 = 500 和 100 的中间色
   - 再填补 800、600、400、200

**编号约定**：100（最浅）→ 900（最深）。9 个色阶容易分割。

**对于灰色**：基础色不重要，同样的流程——从两端开始，填补空隙。

**最重要的是**：在实际设计中使用这些颜色后，相信眼睛做微调。但避免频繁添加新色阶——如果不够自律，还不如没有系统。

---

## 规则 4：不要让明度扼杀饱和度

在 HSL 中，当明度接近 0% 或 100%，饱和度的影响被削弱。创建浅色/深色色阶时，需要增加饱和度补偿：

```css
/* ❌ 同样的饱和度，但浅色看起来褪色 */
color-500: hsl(217, 91%, 50%);  /* 鲜艳 */
color-200: hsl(217, 91%, 90%);  /* 褪色 */

/* ✅ 增加饱和度补偿 */
color-500: hsl(217, 91%, 50%);
color-200: hsl(217, 96%, 90%);  /* 保持鲜艳度 */
```

**基础色已高度饱和（100%）怎么办？** 旋转色相来改变亮度：
- 要让颜色更浅 → 朝最近的亮色相旋转（60°黄、180°青、300°品红）
- 要让颜色更深 → 朝最近的暗色相旋转（0°红、120°绿、240°蓝）
- 色相旋转不超过 20-30°

**示例**：黄色色阶——降低明度的同时朝橙色旋转，深色温暖而丰富：
```css
color-100: hsl(45, 100%, 95%);
color-500: hsl(45, 93%, 58%);   /* 黄色基础 */
color-700: hsl(35, 95%, 45%);   /* 朝橙旋转 */
color-900: hsl(25, 95%, 30%);   /* 继续朝橙旋转 */
```

## 规则 5：灰色不必是灰色

真灰色饱和度为 0%，不包含任何色彩。但实际中，我们认为是灰色的颜色饱和程度相当高。

**给灰色加色温**：
```css
/* ❌ 纯灰——感觉冷、没有个性 */
gray-500: hsl(0, 0%, 50%);

/* ✅ 冷灰——加一点蓝色 */
gray-500: hsl(220, 10%, 50%);

/* ✅ 暖灰——加一点黄色/橙色 */
gray-500: hsl(45, 8%, 50%);
```

**浅色和深色色阶也要保持饱和度**，否则接近 50% 明度的灰色会显得褪色。

---

## 规则 6：对比度与无障碍

WCAG 建议：
- 正文（< 18px）：对比度 ≥ 4.5:1
- 大文字（≥ 18px 或 ≥ 14px 粗体）：对比度 ≥ 3:1

**有色背景上的白色文字**——要满足 4.5:1 对比度，背景需要异常深。解决方案：**翻转对比度**——在浅色背景上用深色文字：

```html
<!-- ❌ 深色背景 + 白色文字 → 对比度可能不达标，且深色块过于抢眼 -->
<button class="bg-blue-700 text-white">提交</button>

<!-- ✅ 翻转：浅色背景 + 深色文字（颜色仍在辅助传达信息） -->
<button class="bg-blue-50 text-blue-900">提交</button>
```

**深色面板中的次要文字**：从背景色开始调整明度/饱和度很难同时满足对比度又区别于白色。解决：将色相朝更亮的颜色旋转（青/品红/黄），在不接近白色的情况下增加对比度。

---

## 规则 7：不要仅依赖颜色传递信息

色盲用户无法仅靠颜色区分信息。每次用颜色传达信息时，同时用其他方式：

```html
<!-- ❌ 只用颜色区分涨跌 -->
<span class="text-green-500">+15%</span>
<span class="text-red-500">-8%</span>

<!-- ✅ 颜色 + 图标 + 符号 -->
<span class="text-green-600">↑ +15%</span>
<span class="text-red-600">↓ -8%</span>
```

对于图表中的多条趋势线：用明暗对比（浅线和深线）而非不同颜色来区分。色盲用户分辨明暗比分辨色相容易得多。

## 检查清单

- [ ] Hex 颜色是否已转换为 HSL？
- [ ] 灰色是否有 8-10 个色阶？主色/强调色各有 5-10 个色阶？
- [ ] 是否用了纯黑（`#000`）？应替换为深灰
- [ ] 浅色色阶是否通过增加饱和度避免了褪色？
- [ ] 灰色是否加了色温（蓝灰或暖灰）？
- [ ] 正文对比度 ≥ 4.5:1？大文字 ≥ 3:1？
- [ ] 有色背景上的白色文字是否考虑翻转对比度？
- [ ] 是否有仅用颜色区分的信息？需要加图标/符号辅助
