# 图片优化规则

图片能成就一个设计，也能毁掉它。本章关注如何让图片在 UI 中发挥最大作用。

---

## 规则 1：用好的照片——别无选择

糟糕的照片会毁掉一个设计。选项：
1. 聘请专业摄影师（需要非常具体的照片时）
2. 使用高质量图库（Unsplash、Pexels 等免费资源）

**永远不要**用占位图然后指望以后用手机拍几张替换——这从来行不通。

---

## 规则 2：背景图片上的文字——保证对比度一致

照片有非常亮的区域和非常暗的区域。单种颜色的文字不可能在整张图片上都可读。

### 方案 A：半透明遮罩

```html
<!-- 黑色遮罩 = 压暗亮区，让浅色文字突出 -->
<div class="relative">
  <img src="hero.jpg" class="w-full h-96 object-cover" />
  <div class="absolute inset-0 bg-black/40"></div>
  <div class="absolute inset-0 flex items-center">
    <h1 class="text-white">标题文字</h1>
  </div>
</div>

<!-- 白色遮罩 = 提亮暗区，让深色文字突出 -->
<div class="relative">
  <img src="hero.jpg" class="w-full h-96 object-cover" />
  <div class="absolute inset-0 bg-white/60"></div>
  <div class="absolute inset-0 flex items-center">
    <h1 class="text-gray-900">标题文字</h1>
  </div>
</div>
```

Tailwind 中 `bg-black/40` = 40% 不透明度的黑色遮罩。

### 方案 B：降低图片对比度

```css
.hero-image {
  filter: contrast(0.7) brightness(1.1);
  /* 降低对比度后可能偏暗，提高亮度补偿 */
}
```

### 方案 C：给图片着色

三步创造效果：
1. 降低对比度 → `filter: contrast(0.8)`
2. 去色 → `filter: grayscale(1)`
3. 叠加品牌色 → 纯色填充 + `mix-blend-mode: multiply`

### 方案 D：文字阴影（发光效果）

```css
.text-on-image {
  text-shadow: 0 0 20px rgba(0, 0, 0, 0.5);
  /* 大模糊半径，无偏移 → 微妙的发光 */
}
```

最佳实践：**组合使用**。例如降低图片对比度 + 文字阴影，可以比只用遮罩保留更多图片动态范围。

---

## 规则 3：一切都有预期尺寸

### 不要放大小图标

在 16-24px 绘制的图标放大 3-4 倍后显得粗笨、缺乏细节。

**解决方案**：将小图标放在带背景色的形状中：
```html
<!-- ❌ 小图标直接放大到 48px -->
<svg class="w-12 h-12">...</svg>

<!-- ✅ 小图标放在带背景色的容器里 -->
<div class="w-12 h-12 bg-blue-100 rounded-lg flex items-center justify-center">
  <svg class="w-6 h-6 text-blue-600">...</svg>
</div>
```

### 不要缩小全尺寸截图

桌面截图缩小 70% → 16px 的文字变成 ~4px，完全不可读。

**解决方案**：
- 用移动端或平板布局截图（文字本来就小，缩小比例小）
- 只截取部分截图
- 或者画一个简化版 UI（移除细节，用小线条代替小文字）

### 不要缩小图标

预期在 128px 下绘制的 Logo 缩小到 16px favicon → 一团糊。

**解决方案**：在目标尺寸重新绘制超级简化版。

## 规则 4：用户上传内容——控制形状和尺寸

### 固定比例容器 + 裁剪

原始宽高比不同的图片会破坏布局：

```html
<!-- ✅ 固定容器裁剪 -->
<div class="aspect-square overflow-hidden rounded-lg">
  <img
    src="{{ userImage }}"
    class="w-full h-full object-cover"
    alt="用户上传图片"
  />
</div>
```

CSS 等价：
```css
.image-container {
  width: 200px;
  height: 200px; /* 固定比例 */
  overflow: hidden;
  border-radius: 8px;
}
.image-container img {
  width: 100%;
  height: 100%;
  object-fit: cover; /* 裁剪而非拉伸 */
}
```

### 防止背景渗透

白色背景图片放在白色 UI 中 → 图片边界丢失。

**解决方案：内阴影或半透明内边框**：

```css
/* 方案 A：微妙的内阴影 */
.user-image {
  box-shadow: inset 0 0 0 1px rgba(0, 0, 0, 0.1);
}

/* 方案 B：半透明内边框（Tailwind ring） */
.user-image {
  @apply ring-1 ring-black/10;
}
```

**不推荐普通边框**：边框颜色容易与图片中的颜色冲突。阴影/半透明内边框更微妙，大多数人甚至注意不到它们的存在。

---

## 检查清单

- [ ] 有没有低质量的占位图或随手拍的照片？
- [ ] 背景图片上的文字是否在所有区域都可读？
- [ ] 如果用了遮罩，是否需要组合使用（降低对比度 + 文字阴影）来保留更多图片细节？
- [ ] 图标是否在预期尺寸下使用（不放大、不缩小）？
- [ ] 用户上传内容的容器是否用了固定比例 + `object-fit: cover`？
- [ ] 用户上传图片是否有防止背景渗透的处理（内阴影/内边框）？
