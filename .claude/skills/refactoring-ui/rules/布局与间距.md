# 布局与间距优化规则

间距是清理设计最强大的工具。好的间距让界面呼吸、干净、易于解读。

---

## 规则 1：先给过多空间，再收回

**反模式**：从 0 开始加 margin/padding，直到「不主动看起来糟糕」。元素只获得不至于难看的最小呼吸空间。

**正确做法**：从过大的值开始，逐步减小直到满意。

```html
<!-- ❌ 默认紧凑，看起来拥挤 -->
<div class="space-y-2 p-3">
  <h2 class="mb-1">标题</h2>
  <p>内容...</p>
</div>

<!-- ✅ 先从过多空间开始，逐步收紧 -->
<div class="space-y-6 p-8">
  <h2 class="mb-4">标题</h2>
  <p>内容...</p>
</div>
```

现实中的经验：聚焦单个元素时看起来「有点过多」的空间，在完整 UI 的上下文中往往就「刚刚好」。

**例外**：仪表盘等需要高密度信息展示的场景可以有意识地做紧凑设计——这应该是主动选择，而非默认结果。

---

## 规则 2：建立间距系统

不要为 120px 和 125px 纠结。使用预定义的间距阶梯。

**推荐间距阶梯**（以 16px 为基础值）：

| 名称 | 值 | 使用场景 |
|------|-----|----------|
| px-0.5 | 2px | 极紧凑分隔 |
| px-1 | 4px | 图标与标签间 |
| px-2 | 8px | 紧密关联元素间 |
| px-3 | 12px | 组件内部小间距 |
| px-4 | 16px | 默认内边距 |
| px-6 | 24px | 卡片内边距、段落间距 |
| px-8 | 32px | 区块间距 |
| px-12 | 48px | 大区块间距 |
| px-16 | 64px | 页面章节间距 |
| px-24 | 96px | 首屏大间距 |

**原则**：
- 小端密集（12→16 = 33% 增长）
- 大端稀疏（500→520 = 4% 增长，几乎不可感知）
- 相邻值差距 ≥ 25%

**Tailwind 可以直接用**：`p-4`、`m-8`、`gap-6`、`space-y-12` 等，Tailwind 的间距刻度本身就是一个好的系统。

## 规则 3：组内间距 < 组间间距

**核心**：间距建立关系。相关元素靠近，不相关元素远离。

**场景 1：表单**
```html
<!-- ❌ 标签与输入框的间距 = 表单组之间的间距 -->
<div>
  <label>姓名</label>
  <input class="mb-6" />  <!-- 6 是到下一个组的间距 -->
  <label>邮箱</label>
  <input />
</div>

<!-- ✅ 标签到输入框 2 < 表单组之间 6 -->
<div class="space-y-6">
  <div>
    <label>姓名</label>
    <input class="mt-1" />  <!-- 组内间距小 -->
  </div>
  <div>
    <label>邮箱</label>
    <input class="mt-1" />
  </div>
</div>
```

**场景 2：文章章节**
```html
<!-- ❌ 标题上方间距不够 -->
<h2 class="mt-4 mb-2">第二章</h2>

<!-- ✅ 标题上方间距 > 标题下方间距 -->
<h2 class="mt-10 mb-3">第二章</h2>
```

**场景 3：水平排列的组件**
```html
<!-- ❌ 各组件间距相同，分不清组 -->
<div class="flex gap-6">
  <span>编辑</span>
  <span>删除</span>
  <span>评论</span>
  <span>分享</span>
</div>

<!-- ✅ 不同功能组之间间距更大 -->
<div class="flex gap-6">
  <span>编辑</span>
  <span>删除</span>
</div>
<div class="flex gap-6 ml-10">
  <span>评论</span>
  <span>分享</span>
</div>
```

## 规则 4：不用填满屏幕

如果内容只需要 600px，就用 600px。给每个元素恰好需要的空间。

```html
<!-- ❌ 强行填满宽屏幕 -->
<div class="w-full flex">
  <aside class="w-3/12">侧边栏</aside>
  <main class="w-9/12">主内容</main>
</div>

<!-- ✅ 侧边栏用固定宽度，主内容弹性 -->
<div class="flex">
  <aside class="w-64 shrink-0">侧边栏</aside>
  <main class="flex-1">主内容</main>
</div>
```

**移动优先策略**：从 400px 画布开始设计移动端，再放到大屏幕上调整。你需要的调整往往比你想象中少。

**列布局**：窄表单放在宽容器中感觉不平衡时，把辅助文字拆到另一列，而不是把表单本身变宽。

---

## 规则 5：用 max-width 而非网格百分比

**问题**：12 列网格把元素设为百分比宽度时，侧边栏会跟着屏幕变宽/变窄，超出合理范围。

```html
<!-- ❌ 网格百分比：侧边栏跟着缩放 -->
<aside class="w-3/12">...</aside>

<!-- ✅ 固定宽度 + max-width -->
<aside class="w-64">...</aside>
```

**卡片场景**：
```html
<!-- ❌ 固定用网格列宽 6/12，中等屏幕可能比大屏幕更宽 -->
<div class="w-6/12 mx-auto">登录卡片</div>

<!-- ✅ 给 max-width，只在屏幕比 max-width 窄时才缩小 -->
<div class="w-full max-w-md mx-auto px-4">登录卡片</div>
```

---

## 规则 6：相对尺寸不可盲目缩放

大屏幕上的比例关系不能直接缩放到小屏幕。

```css
/* ❌ 假设比例不变 */
/* 桌面：正文 18px，标题 45px = 2.5x */
/* 移动端：正文 14px * 2.5 = 35px 标题 → 太大！ */

/* ✅ 各屏幕尺寸独立调整 */
/* 桌面：正文 18px，标题 45px */
/* 移动端：正文 14px，标题 20-24px（约 1.5-1.7x 而非 2.5x） */
```

**组件内也是如此**：大按钮的内边距应该更宽松，小按钮更紧凑——不要用 em 让内边距始终与字号成比例。

---

## 规则 7：避免模糊间距

当两组内容之间的间距 = 组内间距时，用户无法分辨哪些元素是同一组。

**纵向**：表单组间 > 标签到输入框；章节标题上方 > 下方；列表项之间 > 单项内部行高。
**横向**：不同功能组之间 > 组内元素之间。

---

## 检查清单

- [ ] margin/padding 是否用了系统化的间距值（4/8/12/16/24/32/48/64/96）？
- [ ] 任意值（13px、27px、53px）是否已替换为系统值？
- [ ] 组内间距是否明显小于组间间距？
- [ ] 是否有多余的空间填充——元素是否只占据它需要的宽度？
- [ ] 侧边栏等固定宽度元素是否用了固定 px 而非百分比？
- [ ] 响应式字号是否每个断点独立设置，而非机械缩放？
