# 文字排版优化规则

排版是 UI 设计的基石。好的排版让内容易于阅读，差的排版让界面看起来业余。

---

## 规则 1：建立字号层级系统

不使用系统化字号会导致不一致并拖慢工作流。使用手工挑选的固定字号阶梯。

**推荐字号阶梯**：

| 名称 | px | rem | 使用场景 |
|------|-----|-----|----------|
| xs | 12px | 0.75rem | 辅助标签、脚注 |
| sm | 14px | 0.875rem | 次要文字、描述 |
| base | 16px | 1rem | 正文（桌面默认） |
| lg | 18px | 1.125rem | 较大正文、引言 |
| xl | 20px | 1.25rem | 小标题 |
| 2xl | 24px | 1.5rem | 次级标题 |
| 3xl | 30px | 1.875rem | 章节标题 |
| 4xl | 36px | 2.25rem | 页面标题 |
| 5xl | 48px | 3rem | 首屏大标题 |
| 6xl | 60px | 3.75rem | Hero 标题 |

**为什么不用模块化刻度**（如黄金比例 1.618）？
- 产生非整数像素值（31.25px、39.063px），跨浏览器亚像素舍入不一致
- UI 设计中需要的字号比模块化刻度更密集（12→16 之间需要 14，16→21 之间需要 18）
- 手工挑选让你对存在哪些尺寸有完全控制权

**不要用 em 定义字号层级**：嵌套元素的 em 会累积产生不在刻度中的值。坚持用 px 或 rem。

```css
/* ❌ em 累积产生不在刻度中的值 */
.parent { font-size: 1.25em; }  /* 20px */
.child { font-size: 0.875em; }  /* 20px × 0.875 = 17.5px，不在刻度中 */

/* ✅ 始终用 rem */
.text-sm { font-size: 0.875rem; }  /* 14px，始终一致 */
```

## 规则 2：行高

**默认行高 1.5** 是正文的好起点，但不是万能的。

**行高与行宽成正比**：
- 窄段落（~45 字符）→ 行高 1.5
- 宽段落（~75 字符）→ 行高可达 2

原因：眼睛从行尾跳到下一行行首的距离越长，越容易找不到位置。

**行高与字号成反比**：
- 小字（12-14px）→ 行高 1.5-1.6（需要更多帮助找下一行）
- 正文（16px）→ 行高 1.5
- 标题（24-30px）→ 行高 1.2-1.3
- 大标题（48px+）→ 行高 1（不需要额外行间距）

```css
/* Tailwind 实现 */
.text-xs   { line-height: 1.5; }  /* 小字，需要高行高 */
.text-base { line-height: 1.5; }  /* 正文默认 */
.text-xl   { line-height: 1.4; }  /* 小标题 */
.text-3xl  { line-height: 1.2; }  /* 章节标题 */
.text-5xl  { line-height: 1; }    /* Hero 标题 */
```

## 规则 3：行宽

每行 **45-75 个字符** 最佳阅读体验。用 em 单位控制：

```css
/* 每行最多 65 个字符的黄金宽度 */
article p {
  max-width: 30em; /* ≈ 480px at 16px */
}
```

**即使整体布局更宽，段落宽度也要限制**：
```html
<!-- ✅ 主内容区域宽，但段落限制宽度 -->
<main class="max-w-4xl">
  <p class="max-w-prose">这段文字每行不超过 65 个字符...</p>
</main>
```

Tailwind 的 `max-w-prose` 类（65ch）正是为此设计的。

## 规则 4：对齐方式

| 内容类型 | 对齐方式 | 原因 |
|----------|----------|------|
| 长文本（> 3 行） | 左对齐 | 眼睛有固定起点找下一行 |
| 短标题/独立文字块 | 可居中 | 视觉效果好 |
| 数字表格 | 右对齐 | 小数点对齐，便于扫视比较 |
| 两端对齐 | 必须启用连字符 | 避免单词间奇怪的大间隙 |

**混合字号按基线对齐**：
```html
<!-- ❌ 垂直居中——不同字号的基线偏移 -->
<div class="flex items-center">
  <h2 class="text-2xl">卡片标题</h2>
  <span class="text-sm">操作按钮</span>
</div>

<!-- ✅ 按基线对齐——利用眼睛已有的对齐参照 -->
<div class="flex items-baseline">
  <h2 class="text-2xl">卡片标题</h2>
  <span class="text-sm">操作按钮</span>
</div>
```

## 规则 5：字体选择

**UI 设计首选中性无衬线字体**。不确定时用系统字体栈：

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
  "Helvetica Neue", Arial, "Noto Sans", "PingFang SC",
  "Microsoft YaHei", sans-serif;
```

**筛选技巧**：
- 忽略少于 5 种字重的字体家族（字重多的家族制作更用心）
- 正文用字间距宽、x 高度高的字体（小尺寸可读性好）
- 标题可用紧凑字体（标题专用字体）
- 不确定就按流行度排序（Google Fonts → Sort by: Popular）

## 规则 6：字间距（letter-spacing）

**一般情况信任字体设计师**，不动字间距。但有例外：

**收紧标题**：
```css
/* 字间距宽的字体用于标题时，收紧模仿专用标题字体 */
h1 {
  font-family: "Open Sans", sans-serif;
  letter-spacing: -0.02em; /* 略微收紧 */
}
```

**提高全大写文字可读性**：
```css
/* 全大写文字字母高度相同，缺少区分特征 */
.uppercase-text {
  text-transform: uppercase;
  letter-spacing: 0.05em; /* 增加字间距 */
}
```

不要反过来——标题字体即使增加字间距，在小尺寸下也很少表现好。

## 规则 7：链接的强调程度分级

并非每个链接都需要彩色下划线。根据链接在界面中的角色分级：

```html
<!-- 文章中的链接：需要突出的颜色 -->
<p>了解更多请访问<a class="text-blue-600 underline">官方文档</a>。</p>

<!-- 导航/列表中的链接：更微妙的强调 -->
<a class="font-medium text-gray-900 hover:text-gray-600">首页</a>

<!-- 辅助性链接：仅悬停时显示 -->
<a class="text-gray-500 hover:underline">设置</a>
```

## 检查清单

- [ ] 是否定义了字号层级系统（而非到处是任意字号）？
- [ ] 字号是否用 rem/px 定义，没有用 em 嵌套累积的问题？
- [ ] 行高是否根据行宽和字号做了调整（宽段落更高，大标题更低）？
- [ ] 每行是否控制在 45-75 字符？
- [ ] 混合字号是否按基线对齐而非垂直居中？
- [ ] 全大写文字是否增加了字间距？
- [ ] 字重 < 400 是否已替换？
