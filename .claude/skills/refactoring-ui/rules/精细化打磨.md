# 精细化打磨规则

最后的润色让设计从「能用」到「精致」。这些细节累积起来，就是专业与业余的区别。

---

## 规则 1：给默认样式加码

不添加新元素，增强已有的：

### 项目符号 → 图标

```html
<!-- ❌ 普通列表 -->
<ul class="list-disc pl-5">
  <li>端到端加密</li>
  <li>自动备份</li>
  <li>团队协作</li>
</ul>

<!-- ✅ 图标替换项目符号 -->
<ul class="space-y-2">
  <li class="flex items-center gap-2">
    <svg class="w-5 h-5 text-green-500">🔒</svg>
    端到端加密
  </li>
  <li class="flex items-center gap-2">
    <svg class="w-5 h-5 text-green-500">💾</svg>
    自动备份
  </li>
</ul>
```

### 引号 → 视觉元素

```html
<!-- ✅ 放大引号作为装饰 -->
<blockquote class="relative pl-8">
  <span class="absolute left-0 top-0 text-6xl text-gray-300 leading-none">"</span>
  <p class="text-lg italic text-gray-700">这是一段引用的文字...</p>
</blockquote>
```

### 链接 → 自定义样式

```html
<!-- ✅ 品牌色自定义下划线 -->
<a href="#" class="text-blue-600 font-medium underline decoration-blue-300 decoration-2 underline-offset-4 hover:decoration-blue-500">
  了解更多
</a>
```

### 表单控件 → 自定义样式

```html
<!-- ✅ 自定义复选框——选中态用品牌色 -->
<label class="flex items-center gap-2 cursor-pointer">
  <input type="checkbox" class="sr-only peer" />
  <div class="w-5 h-5 border-2 border-gray-300 rounded peer-checked:bg-blue-500 peer-checked:border-blue-500 flex items-center justify-center">
    <svg class="w-3 h-3 text-white hidden peer-checked:block">✓</svg>
  </div>
  同意条款
</label>
```

## 规则 2：用强调色边框增添色彩

在界面平凡的区域加一抹颜色——不需要设计天赋，效果立竿见影。

**场景**：

```html
<!-- 卡片顶部强调线 -->
<div class="rounded-lg shadow border-t-4 border-blue-500 bg-white p-6">
  <h3>专业版</h3>
  <p>适合团队使用</p>
</div>

<!-- 激活导航项侧边强调线 -->
<nav>
  <a class="block border-l-4 border-blue-500 bg-blue-50 px-4 py-2 font-medium">当前页面</a>
  <a class="block border-l-4 border-transparent px-4 py-2 hover:bg-gray-50">其他页面</a>
</nav>

<!-- 提醒消息侧边强调线 -->
<div class="border-l-4 border-yellow-500 bg-yellow-50 p-4">
  <p class="text-yellow-800">请先完善您的个人资料。</p>
</div>

<!-- 标题下方短强调线 -->
<h2 class="text-2xl font-bold pb-2 mb-6 border-b-4 border-blue-500 inline-block">
  功能特色
</h2>

<!-- 页面顶部全局强调线 -->
<header class="border-t-4 border-indigo-600">
  <nav>...</nav>
</header>
```

## 规则 3：装饰背景——不要整个页面一种颜色

### 改变部分背景颜色

```html
<!-- ✅ 不同区块用不同背景色区分 -->
<section class="bg-white py-16">...</section>
<section class="bg-gray-50 py-16">...</section>
<section class="bg-white py-16">...</section>
```

### 轻微渐变

```css
/* 色相差距 ≤ 30° 的渐变最自然 */
.hero-gradient {
  background: linear-gradient(
    135deg,
    hsl(220, 60%, 55%),
    hsl(250, 60%, 55%)
  );
}
```

### 重复图案

```html
<!-- 低对比度背景图案（不干扰内容） -->
<div class="bg-gray-50" style="background-image: url('pattern.svg'); opacity: 0.5;">
  <div class="relative z-10">
    <!-- 页面内容 -->
  </div>
</div>
```

### 简单几何形状/插图

```html
<!-- 单个装饰图形 — 低对比度，不抢内容 -->
<div class="relative overflow-hidden">
  <div class="absolute -top-20 -right-20 w-64 h-64 rounded-full bg-blue-50"></div>
  <div class="relative">
    <!-- 内容在上面 -->
  </div>
</div>
```

## 规则 4：不要忽视空状态

用户首次进入新功能时没有数据——不要让它空着。

```html
<!-- ❌ 空状态——什么都没有 -->
<div></div>

<!-- ✅ 有吸引力的空状态 -->
<div class="flex flex-col items-center justify-center py-16 text-center">
  <img src="illustration.svg" class="w-48 h-48 mb-6 opacity-75" alt="" />
  <h3 class="text-lg font-semibold text-gray-900 mb-2">还没有项目</h3>
  <p class="text-gray-500 mb-6 max-w-sm">
    创建你的第一个项目，开始与团队协作。
  </p>
  <button class="bg-blue-500 text-white px-6 py-2 rounded-lg hover:bg-blue-600">
    创建项目
  </button>
</div>
```

**额外技巧**：在用户创建任何内容之前，把无用的辅助 UI 完全隐藏（如空数据的过滤器、标签页）。

## 规则 5：少用边框——用替代方案

边框是区分元素的好方法，但用太多会让设计感觉拥挤。

**替代方案 A：阴影**
```html
<!-- ✅ 阴影代替边框勾勒轮廓 -->
<div class="shadow rounded-lg p-4">...</div>
```

尤其适合元素背景色与页面背景色不同的情况。

**替代方案 B：两种不同背景色**
```html
<!-- ✅ 相邻区域用不同背景色区分 -->
<div class="bg-white p-4">区域 1</div>
<div class="bg-gray-50 p-4">区域 2</div>
```

如果已经有不同背景色，试试移除中间的边框——可能不需要。

**替代方案 C：增加额外间距**
```html
<!-- ✅ 间距就是最好的分隔 -->
<div class="space-y-12">
  <section>区块 1</section>
  <section>区块 2</section>
</div>
```

## 规则 6：打破常规——不限于既有模式

大多数人对组件有先入为主的认知。跳出思维定式：

**下拉菜单**不一定是白色盒子+链接列表：
- 分成多个区域
- 多列布局
- 添加辅助文字、彩色图标
- 添加分组标题

**表格**不一定是严格的列结构：
- 合并不需要排序的相关列
- 用不同的行布局引入层次
- 在单元格中添加图片、颜色、进度条

**单选按钮**可以不是圆圈+标签：
```html
<!-- ✅ 可选择卡片代替单选按钮 -->
<div class="grid grid-cols-3 gap-4">
  <label class="border-2 rounded-lg p-4 cursor-pointer has-[:checked]:border-blue-500 has-[:checked]:bg-blue-50">
    <input type="radio" name="plan" class="sr-only" />
    <h4 class="font-semibold">基础版</h4>
    <p class="text-sm text-gray-500">适合个人使用</p>
  </label>
  <!-- 更多卡片... -->
</div>
```

## 规则 7：超链接颜色用品牌色

不要在链接中使用蓝色只是因为「链接应该是蓝色的」。用你的品牌主色：

```css
a {
  color: hsl(217, 91%, 60%); /* 品牌蓝，不是浏览器默认蓝 #0000EE */
}
```

---

## 规则 8：Icon 不要同等强调

页面上的图标不需要都一样的颜色或大小。用颜色区分主次：

```html
<!-- ❌ 所有图标同一颜色，全都在争夺注意力 -->
<div class="flex gap-4 text-gray-600">
  <svg>编辑</svg>   <!-- 主要操作 -->
  <svg>删除</svg>   <!-- 破坏操作 -->
  <svg>分享</svg>   <!-- 次要操作 -->
</div>

<!-- ✅ 颜色区分主次和语义 -->
<div class="flex gap-4">
  <svg class="text-gray-700">编辑</svg>
  <svg class="text-red-500">删除</svg>
  <svg class="text-gray-400">分享</svg>
</div>
```

---

## 规则 9：焦点态不可省略

永远不要移除 `:focus-visible` 样式——它对于键盘导航用户至关重要。

```css
/* ❌ 移除焦点态 */
button:focus { outline: none; }

/* ✅ 自定义焦点态，但保留它 */
button:focus-visible {
  outline: 2px solid hsl(217, 91%, 60%);
  outline-offset: 2px;
}
```

Tailwind：`focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2`

## 检查清单

- [ ] 项目符号列表是否可以用图标替换？
- [ ] 表单控件（复选框/单选按钮）是否自定义了样式且选中态用品牌色？
- [ ] 是否有地方可以加一条彩色强调线（卡片顶、导航侧边、标题下）？
- [ ] 背景是否太单调？能否在部分区块用不同背景色或微妙渐变？
- [ ] 空状态是否加了插画/提示/CTA，无用 UI 是否隐藏？
- [ ] 是否为区分元素用了过多边框？能否用阴影/背景色/间距替代？
- [ ] Icon 是否用颜色区分了主次和语义？
- [ ] 链接颜色是否用了品牌色而非浏览器默认蓝？
- [ ] 所有交互元素是否保留了焦点态样式？
