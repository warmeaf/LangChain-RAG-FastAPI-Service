✅ 核心优势（本文代码超出文章预期的部分）

  • 三路图片召回：OCR文本 + CLIP视觉向量 + 多模态LLM描述，完全覆盖文章2.2节三种方案

  • 反馈闭环：贝叶斯平滑 CTR + 文档权重动态更新 + QueryLog 离线分析
  • 动态混合检索权重：根据查询特征（含编号/日期/URL）自动调整 BM25/向量的RRF融合权重
  • PDF视觉去重：感知哈希(pHash)识别重复页面，节省视觉模型调用成本
  • 文档类别自动检测：基于文件名关键词分类并分配预设权重（政策制度1.0/会议纪要0.3等）

⚠️ 待改进领域（按严重度排序）

  • BM25 中文分词缺陷：直接用 .split() 对中文完全无效，是当前最严重的功能性缺陷

  • PDF 缺乏章节结构识别：当前按页处理，未做标题层级/语义边界的段落切分

  • PPT 未按幻灯片逻辑切分：使用 unstructured 通用处理，文章要求保持完整观点
  • 缺少语义边界切分器：文章提到的 NLP 语义分割方案未实现

🏗️ 系统架构对照

📥 文档摄入管道 (Data Ingestion Pipeline)

上传文件 → DocumentTypeRouter
├─ .xlsx → ExcelProcessor (行→NL, 多级表头, 合并单元格)
├─ .py/.js/.ts/.java/.go → CodeProcessor (tree-sitter AST)
├─ .md → HeadingSplitter (标题层级切分)
├─ .pdf → OCRProcessor → pdf_multimodal_loader (视觉模型)
├─ .pptx → unstructured + format_preserver
├─ .docx → unstructured + format_preserver
└─ .txt → 直接加载
→ AsyncTextSplitter (递归+语义合并)
→ bge-large-zh 向量化 (1024维)
→ Milvus 存储 (IVF_FLAT)
→ CLIPImageEmbedder (图片视觉向量，512维)
→ 类别检测+质量评分 → DocWeight 初始化

🔍 检索与排序管道 (Retrieval Pipeline)

用户查询 → QueryProcessor
├─ 压缩 (len>400 → LLM压缩)
├─ 拆解 (→ 最多3个子问题)
└─ 扩展 (→ 每子问题2个变体)
+ HyDE 假设文档生成
→ 混合检索 (BM25 + 向量) + CLIP图片检索
→ RRF 融合 (Top-100, 动态权重)
→ bge-reranker-large 精排 (归一化)
→ MultiFactorRanker (相关性×0.5 + 时间×0.3 + 权重×0.2)
→ LLM 总结生成 (Top-8)
→ QueryLog 记录 + FeedbackService 反馈闭环
