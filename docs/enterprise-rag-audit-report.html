13 实现亮点（超越标准的 6 个方面）

  1. HyDE 假设文档嵌入 — 学术前沿方法，文章未提及。LLM 先生成假设答案，再以该答案作检索变体，显著提升召回率。
  2. CLIP 跨模态图片检索 — 文章提到三种图片向量化方法，当前实现了独立的 CLIP visual collection + 文本→图片跨模态搜索，这是三路召回的核心基础设施。
  3. tree-sitter 代码 AST 切分 — 使用工业级语法解析器（非正则），支持 5 种语言，保留父级上下文。节点级切分比文章的"按函数切分"更精细。
  4. PDF 视觉去重 + 批量处理 — pHash 感知哈希去重 + 批次并发请求，节省视觉模型调用成本。文章未提及此类工程优化。
  5. 贝叶斯平滑 CTR — 反馈学习中采用 (clicks+1)/(impressions+2) 平滑，避免小样本偏差。标准教科书级实现。
  6. 动态混合检索权重 — 根据查询特征（编号/日期/版本号）自动调整 BM25/向量权重，超越文章"可以用分类模型判断"的建议。
