# 待办事项

## max_tokens 截断处理

**现状**：execute 节点 while 循环中，当 `stop_reason == "max_tokens"` 时，当前代码将其视为 `end_turn` 直接退出循环。若 LLM 输出在 `tool_use` JSON 中间被截断，`_extract_tool_calls` 无法解析出完整的工具调用，可能导致不可预期行为。

**影响场景**：回答非常长（接近 384K tokens）时可能触发，发生概率较低。

**修复方向**：
- 检测 `stop_reason == "max_tokens"` 时，给 LLM 追加一条提示消息（如"输出被截断，请继续"），继续循环
- 或者设置 `max_tokens` 足够大（当前 384K 基本覆盖所有场景），作为防御性措施
